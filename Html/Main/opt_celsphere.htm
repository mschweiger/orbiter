Celestial sphere

This page provides configuration options for the display of the celestial sphere background.

Stars rendered as pixels

Orbiter uses the position, apparent magnitude and spectral class information from the Hipparcos star database (ESA Hipparcos mission) to render more than 100 000 bright stars as pixels. The settings in this category determine how the apparent magnitude data are mapped to pixel brightness.

  • A higher value for the Max. app. magnitude setting increases the render intensity for bright stars, but causes the brightness to saturate at that apparent magnitude level.
  • A higher value for the Min. app. magnitude setting increases the render intensity for dim stars, and increases the number of dim stars rendered.
  • A lower value for the Min brightness setting decreases the render intensity for dim stars, but does not decrease the number of stars rendered.
  • The magnitude-brightness mapping should be set to exponential for realistic contrast results.

Stars rendered as image

Orbiter can display a starmap image created from Hipparcos and Tycho data on the celestial sphere. This allows for an improved contrast range compared to the single-pixel rendering.

For best results over all field-of-view settings, turn on both single-pixel and image rendering.

Background map

In addition to a starfield image, Orbiter can display a further background image on the celestial sphere. Orbiter ships with a selection of background images. By default, it displays the Milky Way background generated as a synthetic image with data from the ESA Gaia mission. Alternatively, images from global sky surveys at different wavelength bands (from infrared to gamma ray) can be displayed for special effects.

The intensity of the background image can be adjusted with the slider.
